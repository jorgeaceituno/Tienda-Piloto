# Code Review & Debugging

Cuando me pidas revisar o depurar el código, seguiré estos lineamientos:

1. **Revisión de Errores:** Lo primero es verificar si hay errores de sintaxis, variables no definidas o fallos de ejecución.
2. **Rendimiento (DOM):** Evitar manipulaciones excesivas del DOM (por ejemplo, dentro de ciclos).
3. **Consistencia:** Mantener nombres descriptivos para variables y funciones (camelCase para JS).
4. **Limpieza:** Eliminar `console.log()` innecesarios en versiones finales de producción.
5. **Responsividad:** Asegurarme de que la interfaz se visualice correctamente en pantallas pequeñas (móviles) o en cualquier resolución.
